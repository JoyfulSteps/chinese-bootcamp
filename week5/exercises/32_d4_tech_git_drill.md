## Bài Tập 2: Git Flow — Sắp Xếp Đúng (10 phút)

```
A. 代码合并到主分支
B. 从主分支创建新分支
C. 收到新需求
D. 在测试环境测试
E. 请同事代码审查
F. 在分支上写代码，提交
G. 部署到正式环境，上线
```

<details><summary>🔑 Đáp án</summary>C → B → F → E → A → D → G</details>

## Bài Tập 4: Đọc Hiểu (10 phút)

```
我们团队每两周发布一个新版本。这周是发布周。
我负责的任务已经完成了，代码也提交了。

但是另一个同事的任务还没完成。他说需要再
两天时间。所以我们决定推迟上线。

经理说："除了这个功能以外，其他的都准备好了。
大家先把已经完成的代码合并到主分支，在测试
环境里测试。等那个功能完成了再一起上线。"
```

| # | 问题 | Đáp án |
|---|------|--------|
| 1 | 多久发布一个新版本？ | <details><summary>🔑</summary>每两周</details> |
| 2 | 为什么推迟上线？ | <details><summary>🔑</summary>另一个同事的任务还没完成</details> |
| 3 | 经理让大家做什么？ | <details><summary>🔑</summary>先合并代码到主分支，在测试环境测试</details> |
